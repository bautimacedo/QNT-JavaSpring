## Descripción

Cuando se registra una compra de tipo `EQUIPO`, el sistema debe crear automáticamente el ítem correspondiente en el inventario (Dron, Dock, Batería, Hélice) con estado `NO_LLEGO`. Este estado indica que el equipo fue comprado pero todavía no llegó físicamente. El administrador luego entra al ítem en el inventario y completa los datos faltantes (número de serie, modelo, marca, etc.).

El ítem se crea con los datos mínimos que se pueden inferir de la compra: fecha de compra y nombre/descripción si se proporcionó en `descripcionEquipo`. El resto de los campos (marca, modelo, numeroSerie, etc.) quedan en `null` para que el admin los complete.

Este flujo cierra el TODO `// TODO Logica de Stock e Inventario` que existe en `CompraBusiness.add` y `CompraBusiness.update`.

## PASO 3 — Método privado `crearItemEnInventario`

Agregar en `CompraBusiness` el método privado:

```java
private void crearItemEnInventario(Compra compra) {
    String nombre = compra.getDescripcionEquipo() != null ? compra.getDescripcionEquipo() : null;

    switch (compra.getTipoEquipo()) {
        case DRON -> {
            Dron dron = new Dron();
            dron.setEstado(Estado.NO_LLEGO);
            dron.setNombre(nombre);
            dron.setFechaCompra(compra.getFechaCompra());
            dronBusiness.add(dron);
        }
        case BATERIA -> {
            Bateria bateria = new Bateria();
            bateria.setEstado(Estado.NO_LLEGO);
            bateria.setNombre(nombre);
            bateria.setFechaCompra(compra.getFechaCompra());
            bateriaBusiness.add(bateria);
        }
        case HELICE -> {
            Helice helice = new Helice();
            helice.setEstado(Estado.NO_LLEGO);
            helice.setNombre(nombre);
            helice.setFechaCompra(compra.getFechaCompra());
            heliceBusiness.add(helice);
        }
        case DOCK -> {
            // DOCK requiere site_id NOT NULL; no se puede crear automáticamente sin un site.
            // El admin debe crear el Dock manualmente desde la pantalla de inventario.
            // No hacer nada aquí.
            log.info("Compra de DOCK registrada. El ítem debe crearse manualmente en inventario (requiere Site).");
        }
        case ANTENA_RTK, ANTENA_STARLINK -> {
            // AntenaRtk y AntenaStarlink requieren dock_id NOT NULL.
            // No se pueden crear automáticamente sin un Dock existente.
            // El admin las crea desde el panel del Dock correspondiente.
            log.info("Compra de {} registrada. El ítem debe asociarse a un Dock existente manualmente.", compra.getTipoEquipo());
        }
        case OTRO -> {
            // Sin entidad en inventario para tipo OTRO.
        }
    }
}
```

**Verificación:** al crear una compra con tipoCompra=EQUIPO y tipoEquipo=DRON, se genera automáticamente un registro en la tabla `drones` con estado=NO_LLEGO. Ídem para BATERIA y HELICE.

## Notas

- `Dock` tiene `site_id NOT NULL` en la entidad → no se puede crear con datos mínimos. El flujo para docks es: el admin registra la compra (queda el registro de gasto), y luego crea el Dock manualmente en el inventario asignándole el Site. Si en el futuro se quiere automatizar, habría que hacer `site` nullable.
- `AntenaRtk` y `AntenaStarlink` tienen `dock_id NOT NULL` → misma situación.
- Los items creados automáticamente (DRON, BATERIA, HELICE) quedan con `marca`, `modelo`, `numeroSerie` en `null`. El admin los completa al actualizar el ítem desde la pantalla de inventario una vez que el equipo llegue.
- No linkear el ítem creado de vuelta a la `Compra` con una FK por ahora (se puede hacer en una versión futura). La conexión implícita es por `fechaCompra`.
