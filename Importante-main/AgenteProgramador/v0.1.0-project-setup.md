# v0.1.0 — Setup inicial y estructura base del proyecto

**VERSIÓN:** v0.1.0  
**SLUG:** project-setup  
**DEPENDENCIAS:** ninguna  
**ESTIMACIÓN:** 1h  
**PRIORIDAD:** alta

---

## Descripción

Crear la estructura base del proyecto con las convenciones acordadas. Incluye
configuración del entorno de desarrollo, estructura de carpetas, y archivos de
configuración esenciales.

Este es el punto de partida — todas las versiones posteriores dependen de este setup.

---

## PASO 0 — Contexto previo a leer

> ⚠️ Si venís del AGENTE_WORKFLOW.md, saltear este paso.

Leer sin ejecutar nada:
- `README.md` (si existe)
- `package.json` / `Cargo.toml` / manifiesto principal (si existe)

---

## PASO 1 — Verificar estructura de carpetas

Revisar qué carpetas existen y cuáles faltan según la arquitectura definida en `BLUEPRINT.md`.

```bash
ls -la
```

Crear las carpetas que falten:

```bash
mkdir -p src tests docs scripts
```

**Verificación:** la estructura de carpetas coincide con la definida en `BLUEPRINT.md`.

---

## PASO 2 — Configurar archivos de proyecto

Asegurarse de que existan y sean correctos:

- `.gitignore` apropiado para el stack
- `.editorconfig` con convenciones básicas (indent, charset, etc.)
- `.env.example` con variables de entorno documentadas (sin valores reales)

**Verificación:** los archivos existen y tienen contenido coherente con el stack.

---

## PASO 3 — Verificación de herramientas del stack

```bash
# Adaptar al stack real del proyecto
# Ej para Node.js:
node --version
npm --version
npx tsc --version

# Ej para Rust:
rustc --version
cargo --version
```

Documentar las versiones en `README.md` en la sección "Requisitos previos".

**Verificación:** todas las herramientas requeridas están disponibles y con versiones correctas.
